F-09 · P1 · HTTP 403 authorization refusal is misclassified as deterministic request failure

Confidence: High · Owner boundary: OAS HTTP classification → MASC typed provider policy

Kimi Code의 billing-cycle quota exhaustion 8회가 OAS에서 InvalidRequest(unknown)으로 표면화됐고, MASC는 이 타입을 deterministic_request failure judgment로 올렸다. 같은 provider lane이 base/ramarama에서 반복 호출됐다.

공식 Kimi Code 문서는 해당 응답을 HTTP 403 permission error로 정의한다. OAS의 classify_error에는 401과 402만 있고 403이 catch-all InvalidRequest로 떨어졌다. MASC가 quota 문구를 문자열 매칭하는 것은 경계 위반이다.

F-11 · P1 · Keeper shutdown completes before operator-pause dispatch

Confidence: High · Owner boundary: MASC lifecycle command sequencing

2026-07-11 로그에 stopped cannot accept operator_pause가 10회 있다. 대표 흐름은 dashboard shutdown이 running → draining → stopped를 완료한 직후 동일 command path가 operator_pause를 dispatch하는 것이다. FSM은 올바르게 terminal-state rejection을 반환하지만 command orchestration 순서가 불가능한 transition을 생성한다.

shutdown과 pause를 두 번의 독립 state mutation으로 직렬화하지 말고, operator intent를 typed lifecycle command 하나로 만들고 최종 durable meta/registry state를 한 경계에서 commit해야 한다. rejection을 경고만 하고 성공 응답하는 silent partial success도 금지해야 한다.

수정 상태: MASC Draft PR #24111은 unsafe partial ordering으로 unmerged close됐다.
