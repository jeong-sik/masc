직교 구현 축 v1

A · Semantic capability identity and generated projections

대표: #24032, #23940, #23522

transport name, model-visible name, internal handler, actor identity를 문자열 prefix/alias로 결합한 문제군이다. closed capability ID와 descriptor-owned projection을 SSOT로 두고 외부 MCP·Keeper schema·dispatch를 생성하는 축이다.

G · Domain FSM and verification truth

Goal/Task: #21074, #21078, #22629, #23307, #23886 · Board: #23503, #23504, #23768, #23784, #23889 · Scheduler: #23995, #24026–#24028

각 도메인은 약하게 결합된 typed transition과 durable receipt를 소유하고, 공통 queue/wake substrate 위에 올라간다. LLM 판단은 권한·상태 전이를 대체하지 않는다.

H · Projection and verification must reflect execution truth

대표: #23309, #23310, #23463–#23466, #23507, #23932, #23936, #23960

Dashboard/IDE/CI가 추론된 상태, 실행되지 않은 테스트, 가짜 필드를 성공 증거로 표시하는 문제군이다. projection은 revision/source/error를 포함한 canonical execution snapshot만 소비한다.

B · Durable authority and monotonic recovery evidence

대표: #24049, #24028, #24027, #24030, #23988

decode 가능한 persisted state를 곧바로 실행 권한이나 최신 상태로 간주하는 문제군이다. generation/hash/revocation evidence, semantic reload validation, tombstone/epoch 정책을 projection과 분리해야 한다.

C · Transaction boundary: I/O outside state locks, persist-before-commit

대표: #24025, #24029, #23939

mutable state mutex 안에서 filesystem/network 작업 또는 실패 가능한 callback을 수행해 poison·head-of-line blocking·부분 commit을 만드는 문제군이다. immutable candidate → durable commit → 짧은 in-memory swap 순서를 공통 계약으로 만든다.

D · Immutable runtime selection and typed provider outcomes

대표: #24022, #24023, #23985, #23984

선택된 runtime policy를 이후 ID 재해석으로 찢거나 provider 오류 문자열로 정책을 분류하는 문제군이다. OAS가 wire outcome을 typed catalog로 보존하고, MASC는 한 turn의 immutable resolved runtime snapshot만 소비해야 한다.

E · Per-Keeper continuation, backpressure, and failure isolation

대표: #23939, #23532, #23507

장기 thinking/tool/HITL/poison message가 한 Keeper lane의 finalization과 reactive work를 굶기는 문제군이다. 임의 timeout/cap 대신 typed phase receipt, durable continuation, explicit backpressure와 lane-local recovery를 사용한다.

F · Authentication, subject kind, and provenance

대표: #24031, #24026

Admin role, human subject, dashboard owner, loopback origin을 문자열이나 편의 default로 합성하는 문제군이다. typed subject/origin/permission을 credential에서 끝까지 보존하고 unknown 또는 indeterminate persistence는 fail-closed한다.
