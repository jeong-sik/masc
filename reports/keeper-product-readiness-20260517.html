Keeper Product Readiness Checkpoint

Date: 2026-05-17. Goal: reach a basic product level where keepers do not die silently and repeated silence cannot remain invisible.

Product Invariant

InvariantExpected runtime behaviorStatus in this pass
No silent death Admission denial, crash, watchdog, and dead phases must produce typed lifecycle, metric, or health/operator signal. partially covered Existing supervisor surfaces are strong; open queue still needs PR gate cleanup.
No repeated silence Repeated stay_silent crosses from observation into blocker + recovery stimulus + operator visibility. wired Detector now returns typed outcomes and the turn path enqueues stay_silent_recovery.
No ledger blind spots Unknown reaction kinds and malformed typed payloads degrade KRL summaries instead of disappearing. wired KRL summary now counts unsupported stimuli, unknown reactions, and typed payload parse errors.

Flowcharts

Keeper death and recovery

Boot/TOML
  -> supervise_keepalive
  -> spawn_slots_decision
     -> deny: admission_denied + metric + health/operator blocked
     -> permit: keeper fiber running
        -> heartbeat / turn / receipt
        -> crash, watchdog, or unresolved fiber?
           -> no: continue
           -> yes: sweep_and_recover
              -> clearable within budget: relaunch
              -> not clearable: Dead or Paused with blocker/operator action

Repeated stay_silent

Turn result
  -> speech_act
     -> non-silent: reset streak; clear stay_silent_loop blocker when it was latched
     -> stay_silent: increment streak + gauge
        -> below threshold: observe only
        -> threshold crossed:
           -> metric + warning
           -> registry failure_reason = Tool_required_unsatisfied(stay_silent_loop)
           -> meta blocker_class = stay_silent_loop
           -> enqueue + wake stay_silent_recovery stimulus
           -> KRL records typed stimulus

KRL degraded summary

Reaction ledger rows
  -> summarize
     -> pending stimulus: degraded + operator_action_required
     -> unsupported stimulus kind: degraded + operator_action_required
     -> malformed typed payload: degraded + operator_action_required
     -> unknown reaction kind: degraded + operator_action_required
     -> all stimuli reacted and no degraded rows: ok

SurfaceChangeOperator effect
