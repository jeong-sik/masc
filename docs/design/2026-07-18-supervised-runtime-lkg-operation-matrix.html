3. 메커니즘과 순서

Fork child + private nonce→ Build/Select executable→ Hash exact bytes→ Publish candidate.v2→ exec child→ PID-bound listener proof→ bind-host HTTP health→ Durable LKG + nonce proof

Correctness는 polling 횟수나 대기 시간으로 결정하지 않는다. probe cadence는 관찰 지연만 바꾸며, 승격 조건은 PID·health·hash의 논리곱이다. timeout이 지나면 성공으로 간주하는 경로는 없다.

7. Rollout 순서

  1. focused shell integration suite와 changed-file format/lint를 통과한다.
  2. 빈 runtime root에서 최초 정상 boot 후 LKG descriptor 생성 여부를 확인한다.
  3. 실행 중 bare Dune을 시작하고 runtime을 종료해 exact LKG 복구를 fault-inject한다.
  4. LKG bytes를 변조하고 동일 실험에서 terminal fail-closed를 확인한다.
  5. 24시간 canary 후 startup attempts, listener downtime, false promotion을 집계한다.
  6. 그 뒤에만 10 Keeper × 8시간 mixed-workload product gate에 포함한다.
