# RFC-0228: Paged lane pull + fact-retention harness

Status: Draft · Re-entry of RFC-0223 §5 "digest" — reframed · No summarizer process, no derived store
Drafted by: Claude Fable 5 (v2 follow-up session, 2026-06-11).

## §1 Problem

A keeper can only see the tail of a lane. The pull tool caps at the
store window — `default_limit = 20`, `max_limit = 100`
(`lib/keeper/keeper_surface_read.ml:12-13`), clamped again at the call
site (`lib/keeper/keeper_surface_read.ml:121`) — and the store itself
returns at most the last 100 user/assistant lines
(`lib/keeper/keeper_chat_store.ml:297-298`):

```ocaml
let max_history = 100
let max_total_lines = 400
```

Anything older than the window is unreachable through any keeper
tool: "지난주에 그 채널에서 합의한 것"은 파일에는 있지만 keeper의 도구
표면에는 없다. RFC-0226 P1 ambient recording widens what lands in the
lane, so the gap between "recorded" and "reachable" now grows with
every busy channel.

RFC-0223 §5 deferred a `mode: digest` parameter
(`docs/rfc/RFC-0223-typed-connector-surfaces-presence-pull-speaker.md:206`)
because summarization is non-deterministic and shipping it without an
evaluation harness violates the harness-first rule. That deferral is
correct; what it left unstated is that **most of the value needs no
summarizer at all** — the keeper is already an LLM in a turn; what it
lacks is deterministic access to older pages.

## §2 Principles

1. **Paging is deterministic; digestion is the keeper's own job.** We
   extend the pull surface with an offset, and let the keeper read and
   compress in its own turn. No background summarizer process, no
   materialized digest store, no write-time derivation (owner
   constraint, RFC-0223 §2 principle 6).
2. **No cursors.** `before` is a caller-supplied argument, not server
   state. The server remembers nothing between calls (the unread-cursor
   rejection in RFC-0223 §6 stands:
   `docs/rfc/RFC-0223-typed-connector-surfaces-presence-pull-speaker.md:236`).
3. **Read cost stays bounded.** Each page read is tail-bounded the same
   way RFC-0226 P2 bounded `load` (`keeper_chat_store.ml` §
   `tail_read_bytes`); paging must not reintroduce full-file scans.
4. **Harness before any digest mode.** If a `mode: digest` ever ships,
   its acceptance gate is the fact-retention harness of §4 — measured
   recall, not vibes.

## §3 Design

### 3.1 `keeper_surface_read` gains `before`

```
keeper_surface_read { surface, limit?, before? }
```

- `before`: a message timestamp (the `ts` field already serialized on
  every line). The lane filter returns the window of `limit` primaries
  strictly older than `before`. Omitted → today's behavior (tail).
- Response unchanged plus `oldest_ts` / `has_more` fields so the
  keeper can walk back deterministically: pass `before = oldest_ts`
  to get the next page.
- Store level: a `load_before ~ts` variant that seeks backward in
  bounded byte chunks (reuse `Fs_compat.fold_appended_lines` from a
  computed offset) — never the whole file.

### 3.2 Explicitly NOT in this RFC

- `mode: digest` (LLM summarization inside the tool) — only after §4
  exists and only if paged pull proves insufficient in practice.
- Any persistent digest/index/embedding of lanes.

## §4 Fact-retention harness

Deterministic eval for a non-deterministic skill: can the keeper
answer questions about facts that scrolled past the live window?

- Fixture: synthetic lane JSONL with N planted facts at controlled
  depths (e.g. fact at page 3, decoy variants nearby), generated by a
  seeded script — reproducible.
- Run: ask the keeper (via the normal turn path) questions whose
  answers require paging; harness greps the reply for the planted
  canonical token.
- Metric: recall@depth (facts recovered / planted, bucketed by page
  depth) and tool-call count per recovery. Regression gate: recall at
  the depths the paged pull reaches must be ≥ the floor set on first
  green run.

## §5 Phases

| Phase | Scope | Ships alone? |
|---|---|---|
| P1 | `before` paging (store `load_before` + tool arg + `oldest_ts`/`has_more`) | yes — deterministic, immediately useful |
| P2 | fact-retention harness + baseline | yes — gates any future digest work |

## §6 Workaround self-check (CLAUDE.md signatures)

- Telemetry-as-fix: no. Counter-free.
- String classifier: no — `before` is a typed float ts.
- N-of-M: no.
- Cap/cooldown/dedup/repair: the page-size clamp reuses the existing
  `max_limit` bound; no new caps, no cursor state.
