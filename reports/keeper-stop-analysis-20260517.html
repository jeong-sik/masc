Executive Summary

31 MASC Keeper 정지/비활성/무응답 원인군
24 OAS agent/provider/tool 계층 종료/오류 원인군

4 주요 사이클: supervisor, heartbeat, watchdog, OAS turn

가장 중요한 결론: “keeper가 멈춤”은 단일 상태가 아니다. 코드상으로는 Stopped, Paused, Dead, Zombie, Crashed/Restarting, 그리고 Running이지만 should_run=false 또는 admission/visibility gate로 turn이 스케줄되지 않는 상태가 모두 같은 증상으로 보인다.

가장 위험한 silent 계열은 “정지로 승격되지 않고 turn만 스킵되는 경로”다. FD/disk pressure admission, visibility idle gate, runtime backpressure, semaphore wait timeout, board/event 수집 실패, recurring task auto-disable, OAS callback swallow가 여기에 들어간다.

가장 위험도 높은 단일 항목: OAS ApprovalRequired + callback 없음 = 도구 실행됨 (silent safety bypass). oas/lib/agent/agent_tools.ml:562-584에서 None -> Log.debug "...executing" 분기 확인됨. keeper/runtime profile에서 fail-closed 강제가 단일 root fix. (Suggested Fix Queue §P0 §2)

Silent / Low-Visibility Ledger

WORKAROUND 마킹된 권장은 CLAUDE.md software-development.md §“워크어라운드 거부 기준” 및 RFC-0088 §1 “텔레메트리-as-fix” 안티패턴. counter/이벤트 추가는 *가시화*일 뿐 *fix*가 아니므로 production-blocking 상황 외에는 root fix(RFC) 경로로 흡수해야 한다.

18개 silent/low-visibility 지점. “권장 hardening” 열의 WORKAROUND 표시는 RFC-0088 rejection 대상.

Top Failure Trees

1. Provider/Runtime Failure Tree

provider auth/quota/network/config/capability issue
  -> OAS retry/runtime
    -> retry exhausted or all providers failed
      -> MASC run_keeper_cycle error
        -> transient? no crash count
        -> runtime_exhausted? failure_reason stamped early
          -> streak threshold reached
            -> auto-pause, not restart loop
        -> persistent threshold reached without auto-pause
          -> Keeper_fiber_crash -> restart/dead

2. Host Pressure Failure Tree

low nofile / FD exhaustion / disk pressure
  -> Keeper_fd_pressure or Keeper_disk_pressure admission decision
    -> spawn denied or turn denied
      -> keeper remains registered/alive
        -> no turn scheduled
          -> watchdog may later classify stale, or operator sees silent idle
  -> if actual FD exhaustion error occurs
    -> record_error -> fd pressure circuit trips
      -> subsequent admission blocked/capped

3. Stale/Watchdog Failure Tree

Running keeper with stale last turn/progress
  -> stale watchdog classifies idle/in-turn/mid-turn/noop
    -> sets failure_reason + stop/wakeup
      -> supervisor observes fiber done or forces unresolved crash
        -> if storm/batch/OAS timeout: auto-pause
        -> else restart until budget exhausted
          -> Dead

4. Approval/Tool Contract Failure Tree

required tool expected
  -> OAS completion lacks visible/required tool use
    -> retry feedback if policy configured
      -> exhausted -> CompletionContractViolation
        -> MASC required-tool failure reason
          -> repeated failures -> auto-pause

tool requires approval
  -> ApprovalRequired
    -> callback present: approve/reject/edit
    -> callback missing: executes anyway (silent safety failure)
