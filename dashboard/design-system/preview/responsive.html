This preview mirrors every rule using container queries against a viewport stage you can resize below — no real device required.

1 @media (max-width: 1024px) NONE — no --bp-* in tokens.generated.css .app-shell · .dashboard-header · .header-right laptop fold
2 @media (max-width: 768px) NONE — raw 768px literal 5 grids → 1fr · --font-size-* ramp · .monitor-surface-card · .dashboard-panel tablet · primary fold
3 @media (max-width: 480px) NONE — raw 480px literal .dashboard-header · h1 · .header-subtitle · .tab-pill-bar mobile · narrow phone

Phase 2 candidate: none of the three breakpoints are bound to design tokens. A future --bp-laptop / --bp-tablet / --bp-mobile family would let other stylesheets reference the same fold points without duplicating literals (e.g. @media (max-width: var(--bp-tablet)) once browser support is general). Until then this catalogue is the SSOT.
