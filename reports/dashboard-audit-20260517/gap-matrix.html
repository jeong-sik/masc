Must-Have / Nice-To-Have / Hide

Must-Have

  • Operator Overview with live-vs-fallback state, stale keepers, blockers, approvals, proof gaps.
  • Keeper Detail with lifecycle, heartbeat, goal/task, repo/worktree, latest terminal reason, trace IDs.
  • Intervention Queue with pause/resume/recover/message/approve-deny preview-confirm-recheck flow.
  • Evidence Panel where deterministic proof dominates generated advice.
  • Audit Search across keeper, task, trace, tool, model/provider, approval actor, failure class, timestamp.
  • Code IDE selected-repo `.masc-ide` annotations/regions and decision context.
  • Risk boundaries: capability/tool shard, credential/caller identity, least privilege, high-risk tool state.
  • Unified pending-human-decision queue spanning HITL, verification, goal review, and CDAL.

Nice-To-Have

  • Replay/waterfall trace for a Keeper turn.
  • Topology/causal graph for multi-agent handoffs.
  • Saved operator filters.
  • Benchmark/readiness trends.
  • Autonomy-level history and rollback recommendations.
  • AI-generated summaries below deterministic evidence.

Hide / Remove Candidates

  • Duplicate first-level diagnostic names competing with Monitor's operator story.
  • `Journey Map` unless it becomes the canonical trace replay drilldown.
  • Lab/harness/autoresearch/voice internals from first-screen IA.
  • Typed multimodal internals unless a real artifact gallery and provenance workflow is present.
  • Raw high-volume logs from landing screen.
  • Any green/pass styling for inconclusive proof.
  • Any mutation button without preview-confirm-recheck or copy/run command fallback.
Priorities are code-evidence based and biased toward operator safety.
