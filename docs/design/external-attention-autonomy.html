1. Target Shape

외부 메시지는 transport adapter를 지나면 Discord/Slack/GitHub라는 이름을 잃고, keeper별 pending attention item으로만 동작해야 한다.

Adapter

Discord, Slack, GitHub, dashboard, webhook

External Attention

durable event, dedupe, urgency, actor, preview

Availability

Ready, Busy, Zzz, Offline, Blocked, Degraded

World Observation

reactive turn trigger and prompt section

Keeper Turn

claim, resolve, ignore, or keep pending

같은 content/urgency/target keeper라면 Discord surface_ref를 Slack surface_ref로 바꿔도 scheduler와 availability 결과가 같아야 한다.

3. Data Contract

새 MASC module 후보: Keeper_external_attention. 저장 경로는 live base path 기준이다.

<base_path>/.masc/external_attention/<keeper>.jsonl
type surface_ref =
  | Dashboard of { session_id : string option }
  | Discord of {
      guild_id : string option;
      channel_id : string;          (* channel id or thread channel id *)
      parent_channel_id : string option;
      thread_id : string option;
    }
  | Slack of { team_id : string option; channel_id : string; thread_ts : string option }
  | Github of { repo : string; notification_id : string option }
  | Webhook of { source : string; event_id : string }
  | Gate of { label : string; address : (string * string) list }

type conversation_ref =
  { conversation_id : string; surface : surface_ref }

type external_message_ref =
  { surface : surface_ref; message_id : string; reply_to_message_id : string option }

type urgency =
  | Mention | Direct_message | Ambient | System

type event =
  | Recorded of item
  | Claimed_for_turn of { event_id : string; claim_id : string; turn_id : int option; claimed_at : float }
  | Resolved of { event_id : string; resolved_at : float; reason : string }
  | Ignored of { event_id : string; ignored_at : float; reason : string }

각 item은 event_id, dedupe_key, keeper_name, conversation_ref, optional external_message_ref, actor, urgency, redacted preview, timestamp를 가진다. 현재 pending set은 append-only event log의 projection이다.

dedupe_key는 inbound event id이고 conversation lane key가 아니다. Discord channel과 Discord thread는 둘 다 discord:<guild>:<channel_id> 형태의 conversation id를 쓰며, thread 메시지는 thread channel id가 channel_id가 된다.
Pending semantics are adapter-policy-owned; lifecycle semantics are MASC-store-owned. Discord/Slack/GitHub별 "무엇을 pending으로 볼지"는 adapter boundary에서 결정한다.

4. Availability Model

Ready

실행 가능하고 trigger attention이 pending. 지금 turn으로 승격 가능.

Busy

이미 autonomous/chat/tool turn이 in-flight. 새 attention은 pending으로 남음.

Zzz

실행 가능하지만 pending trigger attention이 없음. sleep/cooldown/poll wait.

Offline

fiber/runtime이 없거나 phase상 실행 불가. recover/start가 먼저.

Blocked

paused, approval, provider exhaustion, continue gate 등으로 실행 불가.

Degraded

명확한 hard block은 아니지만 stale/uncertain 상태. probe 필요.

8. Acceptance Tests

  • Source substitution: Discord and Slack fixtures with same keeper/content produce same scheduler result.
  • Busy queue: autonomous turn holds the slot; external mention does not start another turn and surfaces Busy.
  • Zzz wake: healthy sleeping keeper becomes Ready and wakes on mention/direct message.
  • Blocked preserves pending: paused/runtime-blocked keeper keeps pending attention but does not resolve it.
  • Resolution clears: successful reply moves Claimed to Resolved and prevents repeat reactive turns.
  • Stale claim recovers: crash after claim projects the item back to pending.
  • Restart durability: Pending survives server restart.

9. Invariants

  • NoDroppedAddressedAttention: mention/direct attention cannot disappear without Resolved or Ignored.
  • AtMostOneTurnPerKeeper: pending attention cannot bypass single-flight.
  • SourceNeutralAttention: changing only adapter/surface does not change scheduler decision.
  • BusySurfacesPending: in-flight plus pending trigger attention shows Busy, not Zzz.
  • BlockedDoesNotResolve: blocked/paused/offline cannot mark attention resolved.
