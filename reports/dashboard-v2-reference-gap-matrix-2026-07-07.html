P0 - protocol hardening 먼저

상태를 다른 상태로 접거나, free-form string을 의미판정하거나, 임시 id를 만드는 지점부터 제거한다.

대표: task status fallback, fusion decision substring, connector fallback default, attachment Math.random fallback.

P1 - wake evidence

Scheduler/HITL/Board/Connector가 keeper lane을 깨웠다는 durable event와 UI join key가 필요하다.

UI가 관측 전용이라고 말하는 지점은 정직하지만, 스펙상 wake plane의 끝은 아니다.

P1 - multimodal transport

Keeper chat은 block 구조가 강하지만 board composer는 attachment/voice를 텍스트로 접는다.

Board와 Chat이 같은 multimodal block transport를 공유해야 한다.

PR Slice Order

  1. P0 protocol hardening: task status typed union, fusion decision typed wire, stable attachment ids, connector partial-source disclosures.
  2. Multimodal block transport: share chat/board composer block contract and stop serializing rich media into plain body text.
  3. Scheduler wake evidence: durable runner receipt, lane event, terminal outcome, and dashboard join.
  4. Fusion async surface: typed Judge/Judge-of-Judges graph, sink receipt, and failed-panel isolation without free-form parsing.
  5. Lane per Keeper: one lane timeline per keeper showing busy, queued, deferred, wake cause, current run, and next action.
  6. Board reactive/proactive path: board mention/context inference -> LLM judgment -> wake or queue with explicit ignored reason.
  7. Connector context memory: Discord/Slack/iMessage server/channel/user context as scoped facts and turn source badges.
  8. Memory compaction diff: backend projection for keep/summarize/drop and librarian diagnostic rows.
  9. Visual parity loop: only after truth contracts are stable, run Playwright screenshots against the latest reference bundle.

Do Not Copy As Truth

Runtime/model names in runtime-data.jsx are not current capability truth. Provider/model support changes quickly. Any implementation around Thinking, Reasoning, Tools, Multiturn, Multimodal, or pricing must first read the latest official docs and then bind the result to OAS/runtime schema evidence.

Prototype fixture values must stay out of production: synthetic turn cost/tokens, hardcoded voice transcript/duration, sample Discord messages, and demo model catalogs are design scaffolding only.

Acceptance Rules For Every Slice

Boundary
MASC can consume OAS runtime/provider truth; OAS must not import or assume MASC dashboard concepts.
No silent failure
Malformed, missing, stale, unsupported, and partial data must render as explicit source health or protocol drift.
No fake judgment
Human-like judgment paths go through an LLM boundary or an explicit typed rule. Regex/substrings may extract candidates only, not decide meaning.
Lane isolation
One keeper blocked/busy must not freeze global dashboard behavior or other keepers' lanes.
Immutable UI transforms
Prefer pure transforms over in-place mutation; preserve raw tokens for unknown variants.
Verification
Frontend changes run targeted typecheck/tests. OCaml/dune full build remains CI authority unless explicitly requested.
