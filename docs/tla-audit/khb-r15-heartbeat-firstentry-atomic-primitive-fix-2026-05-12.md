## The drift (sub-class 2: code refactored, citation lags)

The wakeup-consume logic was refactored into `keeper_keepalive_signal.ml` (with `[@@fsm_guard]` PPX annotations explicitly mirroring this spec's actions/postconditions) and `keeper_heartbeat_loop.ml` (`run_smart_heartbeat_gate` dispatcher, `Heartbeat_smart.Skip_idle`). The spec's preamble and `HeartbeatTick` comment hadn't caught up:

1. **Wrong atomic primitive.** `HeartbeatTick` comment said "`[Atomic.exchange wakeup false]` returns true". The runtime is `Atomic.compare_and_set wakeup true false` (in `interruptible_sleep`). `exchange` always succeeds and returns the prior value; `compare_and_set` only succeeds if the value was `true` and returns a bool — so of two racing ticks only one wins the CAS (the other sees `FALSE`). The model is fine (single tick), but the citation named the wrong primitive — and the difference is load-bearing for the *race* the spec's counter is designed to detect (`interruptible_sleep`'s own comment: "a non-zero count means producers are racing the consumer, which is itself a bug class").
2. **Missed the annotated correspondence.** `keeper_keepalive_signal.ml`'s `post_wakeup_signal` (`[@@fsm_guard "Atomic.get wakeup = true"]`) ~ `WakeupSignal`, `post_heartbeat_tick` (`[@@fsm_guard "Atomic.get wakeup = false"]`) ~ `HeartbeatTick`, and `interruptible_sleep`'s inline comment ("Spec: KeeperHeartbeat.tla HeartbeatTick action — wakeup is consumed (TRUE -> FALSE) ... without the discriminator the [Skip_idle] branch would consume the CAS and then skip the cycle (the [MissedWakeup] bug-action)"). The spec preamble didn't point at any of this — so a reader couldn't find the OCaml side's runtime-enforced mirror of these postconditions. Same situation as iter 87 (OPB) where `keeper_stale_watchdog.ml` had a note the spec hadn't picked up.
3. **Imprecise file/function.** `run_heartbeat_loop` is `keeper_heartbeat_loop.ml`@1588 (re-exported via `Keeper_keepalive`'s `include Keeper_heartbeat_loop`), not `keeper_keepalive.ml` directly; the wakeup *consume* is `interruptible_sleep` in `keeper_keepalive_signal.ml`; the *dispatcher* is `run_smart_heartbeat_gate`@1376 in `keeper_heartbeat_loop.ml`.

**Fix (comment-only)**: rewrote the preamble's "Runtime entities modelled" block to name the correspondence anchors (`keeper_keepalive_signal.ml` with the `[@@fsm_guard]` PPX + `interruptible_sleep`; `keeper_heartbeat_loop.ml` with `run_heartbeat_loop` / `run_smart_heartbeat_gate` / `Heartbeat_smart.Skip_idle`; `keeper_keepalive.ml` re-exports + the `Atomic.set entry.fiber_wakeup true` setter) and corrected the atomic primitive (`Atomic.compare_and_set wakeup true false`, with a note on why the CAS semantics matter). Updated the `HeartbeatTick` and `MissedWakeup` comments to name `interruptible_sleep` / `run_smart_heartbeat_gate` / `Skip_idle` and the `[@@fsm_guard]` runtime mirror. Added an iter-90 re-verification date.
