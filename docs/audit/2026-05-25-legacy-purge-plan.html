MASC Legacy Purge Plan

Snapshot: 2026-05-25 12:43:57 KST. Scope: Keeper, Cascade, Tools, Memory surfaces in masc-mcp.
Goal
Break compatibility intentionally where the compatibility layer now slows development or hides runtime truth.
Rule
Delete aliases, duplicate wire keys, compatibility facades, and tests that preserve obsolete behavior. Replace tests with absence checks only when the contract still matters.
Branching
Small draft PRs per axis. Do not batch broad Tool_result or Memory schema breaks with Cascade/OTel cleanup.

P0 Cascade_legacy_runner Misnames the current cascade observation/audit actor as a legacy runner and spreads misleading dotted module references through Keeper, MCP server, dashboard tools, and tests. Rename to Cascade_observation; do not preserve an alias module. Keep behavior tests under the new module. Delete only tests whose assertion is old-name preservation.
P0 OTel legacy duplicate attributes Dual-emitted legacy keys create two sources of truth for the same span fact and keep dashboards coupled to obsolete names. Emit canonical gen_ai.* plus MASC extension keys only. Replace preservation tests with absence tests.
P0 json_ok / json_error aliases in local runtime tools They are explicitly kept for include-cascade compatibility and force lint exceptions to preserve an obsolete helper vocabulary. Delete aliases from Tool_local_runtime_core; call Tool_args directly. No new test needed; static grep plus changed-file build is enough.
P1 Tool_result.legacy_message The field name says tuple-era compatibility even though many current tests and handlers still use it as the primary text channel. Introduce a canonical field name and migrate tests/handlers in batches, then delete legacy_message. Remove tests that parse legacy_message only to keep string-body compatibility. Keep tests that validate user-visible message content through the new field.
P1 Keeper_types compatibility facade Broad re-export surface lets old type paths survive and hides module ownership. It also makes grep-based ownership audits noisy. Move active callers to owner modules, then shrink the facade to documented public types or delete leaf aliases. Delete facade-preservation tests; add owner-module import tests only where public API matters.
P1 Memory/context legacy read fallbacks Fallback readers such as legacy content arrays, checkpoint sidecars, and dated-store fallback hide whether current writers are clean. Split into runtime-data migration and code deletion. Delete read fallback after migration proof or if runtime reset is acceptable. Remove tests that require old-row readability. Keep corruption/drop tests only for current schema.
P2 Cascade deprecated profile names The closed deprecated-name set and metrics ratchet keep accepting old configuration vocabulary. Reject old names at config load and delete the deprecated-profile filter metric after catalog fixtures are current. Turn compatibility tests into rejection tests.
P2 Dashboard and docs comments that describe already-deleted facades They make readers think a compatibility path still exists and cause false positives in audits. Rewrite comments as current contract notes or remove them. No runtime tests; doc grep is enough.
