CROSS-CUTTING

IDE Backbone

모든 12 zone에 공통으로 적용되는 필터·attribution·navigation 규약.

MASC는 단순 dashboard가 아니라 IDE다. 모든 zone은 다음 두 축으로 항상 좁혀질 수 있어야 한다 — ① branch (현재 작업 트리·worktree·git ref) 와 ② keeper (어떤 agent의 활동인가).

① Branch Selector 상단 chrome에 상시 노출. main · release-0.42 · feat/keeper-clarity · worktree-sangsu … 현재 branch에 따라 모든 zone의 데이터 필터, IDE 트리/diff/PR 표시가 동기화.
② Keeper Selector Sidebar의 keeper 클릭 = 전 zone에 keeper filter 활성. composer mention·task claim·cost·decisions 모두 해당 keeper로 좁혀짐. multi-select 지원 (Cmd+Click).
③ Cross-link 규약 모든 ID — task / goal / post / cascade / decision — 는 클릭 → drawer/inspector 연다. IDE side에서 PR·commit·file 클릭은 board/audit으로 reverse-lookup 가능해야 한다.

이 backbone은 topbar에 branch dropdown 추가, sidebar에 keeper multi-select 모드 추가, 모든 신규 zone이 {branch, keepers[]} 컨텍스트를 prop으로 받음 — 세 가지 변경으로 구현된다.

Acceptance Criteria

  • 실데이터 충실도 — 시드 데이터는 모두 .masc/ 실제 row에서 발췌. 임의 placeholder 금지.
  • IDE Backbone 적용 — 모든 신규 zone이 branch + keeper(s) 컨텍스트를 받고 가시적으로 반영해야 함.
  • System fidelity — 모든 zone은 기존 tokens / primitives / icon-less 규약을 위반하지 않음 (no emoji, mono-first, hairline borders, brass-only-for-active).
  • Variant 수 — zone당 평균 2~3 variant; 12 zones × ~2.5 ≈ 30 artboards.
  • Cross-link — 모든 ID(task / goal / post / cascade / decision)는 클릭 가능한 reference로 표현.
  • Hub 갱신 — preview/index.html에 Phase 2 섹션 4개 추가, 기존 카드와 동일한 형식.
  • 문서 — README "Index" 섹션과 SKILL.md 컴포넌트 매니페스트에 12개 신규 zone 등재.
