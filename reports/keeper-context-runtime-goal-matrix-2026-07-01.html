Orthogonal Radius

Part Why It Is Orthogonal Failure Mode To Look For Goal Candidate
Memory / checkpoint Persistence can outlive the runtime that produced it. Large-window checkpoint resumes into smaller-window runtime without pre-dispatch reduction. Runtime-window-aware checkpoint restore and compact.
Compaction policy MASC pre-dispatch and OAS provider compaction are separate layers. MASC ratio below threshold while OAS provider ratio is over 1.0. Shared window evidence and provider-prep token telemetry.
OAS builder/resume OAS has its own provider-capability window resolver and reducer construction. MASC passes only compaction ratio while OAS derives a different context window. Explicit context-window contract or terminal mismatch check.
Turn lifecycle Retries and reroutes happen after the first turn setup. Prepared context budget is stale after runtime rotation. Rebudget on every selected runtime transition.
Manual/direct turns They can bypass the unified turn setup path that recently got the most budget hardening. First direct dispatch uses projection/override budget and exceeds selected runtime cap. One resolver and one budget record for unified, manual, and retry paths.
Runtime catalog Runtime TOML, model rows, provider capability manifest, and OAS catalog can disagree. Provider-qualified id does not resolve but fallback runtime budget is accepted. Catalog drift check with fail-closed assignment validation.
Context injection Temporal/tool context can be injected after MASC estimates input size. Extra system context or post-tool injection pushes provider request over limit. Size caps and token attribution per injected segment.
Recovery / pause Recovery behavior can be correct while root cause remains active. Auto-resume loops repeatedly hit the same provider limit. Pause metadata includes root budget mismatch fingerprint.
Dashboard / health Operators debug through health snapshots, not raw OAS internals. Fleet blocked/paused status hides budget drift. Expose budget source, provider limit, compact actions, and mismatch warning.
Tests / fixtures Existing unit tests cover pieces, not the cross-layer historical failure. Routed runtime test passes but unresolved-provider fallback still regresses. End-to-end small-window resume and log replay fixtures.
