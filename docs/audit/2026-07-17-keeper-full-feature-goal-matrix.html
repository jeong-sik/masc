변하지 않는 네 가지 법

Activity First

한 Provider·Tool·Gate·Memory·child run이 기다려도 같은 Keeper는 다른 runnable work를 수행하고 peer lane은 독립 진행한다.

LLM Decision Boundary

relevance·risk·summary·completion 같은 의미 판단은 configured LLM. 문자열·점수·위험 등급 휴리스틱으로 대체하지 않는다.

Single Writer

durable operation/journal commit이 truth. Event_bus·Raw_trace·OTel·dashboard는 cursor-backed projection이며 독립 settlement writer가 아니다.

Hard Cut

replacement가 증명되면 legacy 코드·test·env·parser·dual writer를 함께 삭제한다. 이름만 바꾼 compatibility layer는 만들지 않는다.

처음부터 만들려던 제품 범위

Keeper & Lane

무한히 살아 있는 persona/worldview agent. Keeper별 durable drain, busy 중 다른 일, exact wake와 continuation.

Board · Goal · Task

약결합 satellite. mention/relevance/completion은 LLM 판단 뒤 exact owner wake. Task closure는 Judge receipt.

Gate · HITL

risk hierarchy가 아니라 exact Always Allowed, configured Auto Judge, nonblocking Manual. 한 lane의 대기가 fleet stop이 아님.

Scheduler

due occurrence와 lane wake를 소유. iCalendar UID/RECURRENCE-ID/RRULE/RDATE/EXDATE/timezone/misfire. schedule 자체는 grant가 아님.

Connector

Discord/Slack 등 connector·space·channel·actor·event provenance 분리. busy reply, durable outbox, lane-local requeue.

Fusion

heterogeneous panel, quorum, Judge-of-Judges, async join. completion/cancel은 정확히 한 terminal과 owner Keeper wake.

Memory · Compaction

기억·망각·요약은 LLM. active Tool identity와 unresolved progress는 exact. checkpoint는 target-fit 뒤 durable 재주입.

Runtime · Multimodal

Text/Image/Voice/Audio provider/model catalog, typed capability와 ordered failover. vendor/model/URL string 추론 금지.

Any as a Tool

Tool, Model, LLM Agent, Keeper, Fusion, Connector, Scheduler를 immutable Any[]/AsyncAny[]로 조합.

Dashboard Chat

Thinking·Reasoning·ToolUse·Progress·ToolResult·multimodal·child run·compaction·reinjection을 causal order로 표시.

OAS Boundary

작고 범용적인 finite Agent SDK. Provider/Model/run/tool occurrence만 소유하며 Keeper·Gate·Board·Fusion을 모른다.

Scale

1/100/1000 혼합에서도 owner identity와 operation law가 동일. 숫자는 관측·capacity이지 correctness admission이 아니다.
