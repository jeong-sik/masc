Code IDE Wiring Details

Implemented path:
  repo select -> activeRepositoryId
  -> fetchWorkspaceTree(repo_id)
  -> fetchWorkspaceFile(repo_id)
  -> fetchGitBlame(repo_id)
  -> fetchGitDiff(repo_id)
  -> editor/diff/ownership render

Repaired metadata path:
  activeRepositoryId exists in coordinator opts
  -> api/ide.ts IdeApiOptions carries keeper + repoId
  -> fetchIdeAnnotations / fetchIdeRegions / create / delete append repo_id
  -> server resolves selected repository base
  -> selected repo `.masc-ide` can show

Repaired region store path:
  producer writes .masc-ide/regions.jsonl
  reader reads .masc-ide/regions.jsonl
  -> generated regions are visible through the existing HTTP reader

Repaired decision trace path:
  keeper .decisions.jsonl
  -> keeper_decisions_json emits choice + reason + context
  -> dashboard.ts decodes KeeperDecision.context
  -> decision-log-trace-bridge preserves choice/reason/context
  -> overlay focus and editor line chips show the decision at file/line

Bonsai Split

`dashboard_bonsai` is a separate OCaml/Bonsai dashboard under `/dashboard/b/*`, while the main Preact dashboard is `/dashboard/*`. Treat it as a parallel/experimental surface unless product owners explicitly promote it into the canonical operator console.

Evidence: `dashboard_bonsai/README.md`, `dashboard_bonsai/src/route.ml`, `server_routes_http_routes_frontend.ml:115-164`.

Audit note: Bonsai contains multimodal files, but the main dashboard source tree had no `*multimodal*` frontend files. Do not count Bonsai-only multimodal UI as complete main-dashboard IA.

Report reflects the local implementation patch in this worktree.
