Activity First

Decision Boundary

Everything Observed

Journal commit이 truth. Event_bus, Raw_trace, OTel, dashboard는 cursor-backed projection이다.

Hard Cut

replacement가 증명되면 legacy 구현·test·env·parser·dual writer를 함께 삭제한다.

처음부터 만들려던 제품 범위

Keeper & Lane

무한히 살아 있는 persona/worldview agent. Keeper별 durable drain, busy 중 다른 일, exact wake와 continuation.

Board · Goal · Task

약결합 satellite. mention/relevance/completion은 LLM 판단 후 wake. Task completion은 schema-valid Judge receipt.

Gate · HITL

risk hierarchy가 아닌 exact Always Allowed, configured Auto Judge, nonblocking Manual. 매 effect마다 fresh decision.

Scheduler

due occurrence와 lane wake를 소유. iCalendar UID/RECURRENCE-ID/RRULE/RDATE/EXDATE/timezone/misfire. schedule 자체는 grant가 아님.

Connector

Discord/Slack 등 connector·space·channel·actor·event provenance 분리. busy reply, durable outbox, lane-local requeue.

Fusion

heterogeneous panel, quorum, Judge-of-Judges, 비동기 join. 완료는 owner Keeper를 깨우며 기다리는 동안 Keeper는 계속 활동.

Memory · Compaction

기억·망각·요약의 의미는 LLM. active Tool identity와 unresolved progress는 exact. compacted checkpoint는 durable 재주입.

Runtime · Multimodal

Text/Image/Voice/Audio provider/model catalog, typed capability와 ordered failover. 이름·URL·vendor string 추론 금지.

Any as a Tool

Tool, Model, LLM Agent, Keeper, Fusion, Connector, Scheduler를 Any/Any[]/AsyncAny[]로 조합.

Dashboard Chat

Thinking·Reasoning·ToolUse·Progress·ToolResult·multimodal·child run·compaction·reinjection을 causal order로 표시.

OAS

작고 범용적인 finite Agent SDK. provider/model/run/tool occurrence만 소유하며 Keeper·Gate·Board를 모른다.

Scale

1/100/1000 혼합에서도 타입과 owner identity가 동일. 숫자는 capacity 관측이지 correctness admission이 아니다.

Compaction 현재 판정: Structural Safety = YES · Overflow Recovery = NO

현재 main에서 실제로 배선된 사실

  • masc_keeper_compact가 target Keeper durable queue에 exact manual request를 넣고 owner heartbeat가 lease한다.
  • typed ContextOverflow만 overflow trigger가 되며, 성공 뒤 exact source stimulus를 requeue한다.
  • configured structured-judge Runtime, Keeper chat Runtime 순으로 declared candidate failover를 수행하며 default Runtime을 만들지 않는다.
  • #25046이 Tool cycle/protected suffix partition, malformed fail-closed, silent pair-repair 삭제와 exact source CAS를 착지했다.
  • checkpoint save, manifest, durable Before/After structural evidence, next-turn checkpoint load와 dashboard reinjection correlation 경로가 있다.
  • peer Keeper는 독립 lane이므로 한 Keeper의 compaction LLM 대기 때문에 fleet가 정지하지 않는다.

이 항목은 real wiring 증거지만, 실패한 Provider 요청을 실제로 fit시키는 완료 영수증은 아니다.

완료를 막는 실제 공백

  • P0 on main: Structurally_unchanged/no eligible/invalid plan에 durable terminal이 없다. Heartbeat가 같은 active lease를 무조건 재큐잉하므로 해당 Keeper의 뒤 stimulus가 영구 기아될 수 있다. Draft #25221→#25223이 기존 event-queue settlement WAL에 source-bound No_compaction을 추가하지만, 병합 및 full stacked CI 전에는 해결로 계산하지 않는다.
  • 선택/실패 Provider의 MaxContext를 결과 admission에 사용하지 않고, 한 번이라도 serialized bytes가 작아지면 성공으로 판정한다. 따라서 overflow 뒤에도 다시 overflow할 수 있다.
  • 모든 eligible Assistant text를 한 provider request에 싣는 one-shot 구조라 compaction request 자체가 overflow할 수 있다. multi-wave/re-measure/convergence가 없다.
  • Closed Tool cycle, open ToolProgress suffix, Thinking/Reasoning/Image/Document/Audio와 metadata-bearing text는 안전하게 보존되지만 compact 불가라 Tool-heavy history는 줄지 않을 수 있다.
  • manual/overflow 모두 LLM 호출 동안 owner turn admission을 쥔다. peer는 살지만 같은 Keeper는 다른 runnable work를 할 수 없다.
  • 재주입은 next-turn loader/dashboard 상관 경로까지 있고 source CAS도 있지만, durable async operation·process restart claim·terminal reinjection receipt E2E는 없다.
  • OAS #2683은 이제 Draft이고 Tool effect authority를 실제 Agent 경로에 연결했지만, 3,176 changed lines이며 handoff child scope와 public persisted-locator resume가 남아 있다. MASC release pin은 여전히 v0.216.3이다.
