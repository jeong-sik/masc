Channel-Independent Turn Surface Plan

Discord는 입력 채널일 뿐이다. Keeper가 자율 턴을 수행 중이면 모든 surface가 같은 `Busy`를 보여야 하고, 아무 턴도 없고 deterministic cycle을 기다리면 `Zzz`를 보여야 한다.

Status: design proposal · Created: 2026-06-11 · Companion MD: docs/design/channel-independent-turn-surface-plan.md
turn owner Busy

RFC-0225 admission slot has an owner. Lane can be autonomous, chat, board, MCP, or operator.

alive, waiting Zzz

Keepalive is healthy, no turn is in flight, and the next autonomous/reactive trigger has not opened.

admissible Idle

Direct work can start now. This is not a connector state and does not depend on Discord.

human action Attention

RFC-0150 attention signal exists. The operator should inspect or unblock something.

not runnable Offline

Runtime, keepalive, or health says the keeper cannot currently execute.

Decision

MASC exposes one keeper-owned turn_presence read model. Channel adapters produce typed inbound events and optional source metadata; they never decide the primary availability label.

Boundary: OAS remains responsible for provider/model execution, hooks, tool-use mechanics, and transport. MASC owns keeper admission, runtime presence, and dashboard/operator projection.

Derivation Order

PriorityInputAvailabilityReason
1runtime missing, keepalive stopped, health offlineOfflineNo live execution surface.
2RFC-0150 attention signal presentAttentionHuman action is blocking useful progress.
3RFC-0225 admission slot has an ownerBusyA turn is currently in flight.
4keepalive healthy, no turn, cooldown/no salienceZzzKeeper is alive and waiting.
5keepalive healthy, direct turn can start nowIdleDirect work can be admitted.
Design point: `Busy` wins over channel origin. A Discord message received during an autonomous turn does not make the keeper Discord-active; the surface still shows `Busy · autonomous`.

Data Flow

Channel Adapter

Discord, Slack, dashboard chat, board, or MCP emits a typed inbound event with optional source metadata.

source only

Turn Admission

RFC-0225 single-flight owner decides whether a turn is admitted, queued, or skipped.

authoritative Busy source

Turn Presence

Keeper_turn_presence derives Busy, Zzz, Idle, Attention, or Offline.

closed sum read model

Surface

Keeper Operations list/detail renders the same availability regardless of which channel woke the keeper.

operator-facing label

Streaming And Typing Reference

Hermes Agent is a useful surface reference: its gateway accepts platform messages, runs a shared agent loop, buffers stream deltas, and progressively edits one target-platform message. Its Discord adapter also keeps typing alive with an 8-second refresh loop while work is in progress.

Typing

Discord typing is a short-lived connector projection. MASC should refresh it only while an accepted inbound is waiting for keeper output.

wait phase only

Streaming Out

Progressive output should be an edit-loop transport: create a reply, coalesce deltas, patch the message, and finalize the visible text.

future transport slice

Turn Presence

Busy, Zzz, and Idle still come from keeper runtime/admission state, not from Discord typing or edits.

MASC-owned SSOT

[근거] Discord typing expires after 10 seconds and returns 204 on success; Discord edit-message supports editing previously sent content; checked 2026-06-11 Asia/Seoul. Hermes README, architecture docs, stream consumer, and Discord adapter show the gateway/edit-loop/8-second typing pattern.

Surface Placement

  • GET /api/v1/keepers/:name/composite adds full turn_presence.
  • Keeper Operations list gets compact fields: availability, lane, reason, started_at.
  • Dashboard SSE emits keeper_presence_changed on kind or owner changes.
  • Connector health remains separate from keeper turn presence.

Done Criteria

  • sangsu autonomous long turn shows Busy in Keeper Operations with Discord disabled.
  • After release, the same surface shows Zzz or Idle based on direct-message admissibility.
  • Discord connector health can be down while keeper turn presence remains truthful.
  • No OAS public schema or provider/model code changes are required.
