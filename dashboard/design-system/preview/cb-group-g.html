cb-group-g · foundational primitives

Empty · Loading · Error · Pagination · Breadcrumb

Cross-cutting primitives every cockpit surface needs but were not bundled in cb-group-a..f / h..k. Each variant is self-contained — no MASC_DATA dependency — and follows SPEC §5 a11y patterns: empty/loading announce via role="status", errors via role="alert", pagination uses a labelled <nav>, breadcrumb is an <ol> with aria-current="page".
