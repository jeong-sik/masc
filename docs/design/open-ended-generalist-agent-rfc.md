# Withdraw open-ended curriculum policy

The proposed automatic curriculum inferred new Tasks/Goals from activity
signals, assigned verifier requirements, and created an approval state outside
the Task/Goal SSOT. That is a heuristic policy subsystem and is withdrawn.

A Keeper may still ask the configured LLM to propose a Task or Goal from typed
observations. Creation then uses the ordinary versioned Workspace API. Advice
artifacts have no transition, runtime, Tool, or Keeper lifecycle authority.
