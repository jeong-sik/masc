Keeper Repository Claim HITL Plan

Unknown keeper repository access should not silently become authority, but it should also not dead-end as a plain reject when the repository identity is known and the only missing fact is operator authorization.

Date: 2026-07-06 Repo: jeong-sik/masc Boundary: MASC only OAS impact: none

Decision

Keep the strict repo access gate as the default authority. Add an explicit HITL claim path for the narrow case where a repository is registered and identity-validated, but the keeper's selected mapping does not include it.

Allowed

Existing mapping permits the repository, or the current default registered-repository scope applies.

HITL Pending

Repository identity is valid, but the keeper mapping excludes it. Queue an approval request and do not block the keeper fiber.

Denied

Repository is unregistered, mapping file is malformed, repository store load failed, or identity mismatched. No claim request is made.

Implementation Slice

  • Add a typed access decision in Keeper_repo_mapping so callers can distinguish not-in-mapping from load/store/identity errors.
  • Add a keeper-side HITL bridge that calls Keeper_approval_queue.submit_pending for repository-claim requests.
  • On operator approve, update keeper_repo_mappings.toml through the existing save_mapping SSOT.
  • On reject or edit, persist no mapping change and emit an explicit log path.
  • Wire read/write filesystem paths to return a structured approval_pending response instead of a plain policy reject for the claimable case.

Non Goals

  • No OAS API or provider behavior change.
  • No automatic repo registration for unregistered or identity-mismatched paths.
  • No heuristic matching from free text, branch names, or command strings.
  • No background mutation without an approval decision.

Validation

Case Expected Result Check
Keeper mapping excludes registered repo HITL pending approval is created Focused Alcotest for repo claim bridge
Operator approves claim Mapping file gains the repository id through save_mapping Resolve pending approval and reload mapping
Unregistered repo id Fail closed; no HITL request Repo mapping decision test
Malformed mapping or repository store Fail closed with explicit diagnostic Existing fail-closed tests remain unchanged
