개발 개선 문서

대규모 리팩토링은 코드 이동보다 먼저 실행 순서의 contract를 고정해야 합니다. 아래 순서는 wire JSON, task/goal state, keeper turn receipt, dashboard read model을 깨지 않기 위한 개발 단위입니다.

1. MCP request context reducer

  • 입력: method/path/headers/body/profile/base_path
  • 출력: session id, accept mode, actor, auth token, internal keeper flag, response framing
  • 목표: H1/H2 중복 제거. Transport는 read/write adapter만 유지.
  • 보호 테스트: H1/H2 parity, unknown session, protocol mismatch, Accept rejection, dashboard actor rewrite, session DELETE, streaming capability.
  • 현재 divergence: H2는 GET SSE branch, known-session validation, streaming POST/SSE framing이 H1과 다릅니다.

2. Tool execution identity state machine

  • 입력: _agent_name, legacy agent_name, auth token, session cache
  • 출력: resolved agent, token owner, keeper identity, strict-auth verdict
  • 목표: mcp_server_eio_execute.ml의 identity/auth/join path를 dispatch에서 분리.
  • 보호 테스트: bearer mismatch, generated alias, internal keeper token, join guard.

3. Keeper turn pipeline contract

  • 입력: meta, observation, channel, selected item, shared context
  • 단계: SpawnAdmission -> EventIntake -> PhaseGate -> PromptPlan -> ProviderAttempt -> ReceiptClose -> PostTurn -> MetricsClose
  • 목표: keeper_unified_turn.ml을 상태 record 전달 방식으로 분해.
  • 보호 테스트: spawn denial no-fork, event queue payload, manifest order, side-effect retry suppression, receipt authority, passive-loop detector, TLA correspondence.

4. Task and Goal pure reducers

  • Task reducer: status/action/actor/version/configured LLM verdict -> decision
  • Goal reducer: phase/action/actor/policy/approval flag -> decision
  • 목표: tool handler와 persistence를 얇게 만들고 상태 전이를 한 곳에서 검증.
  • 보호 테스트: TaskLifecycle TLA, goal quorum, exact assignee preconditions, LLM verdict unavailable/reject/pass, and backlog write failure recovery.

5. Dashboard read-model module split

  • Shell, keeper, execution, OAS telemetry, config, LSP를 projection module로 분리
  • 목표: router module은 route wiring만 하고 JSON projection은 domain read model이 소유.
  • 추가 contract: DashboardSurface envelope로 cache state, stale fallback, source, broadcast hook을 통일.
  • Migration rule: 기존 root JSON field는 유지하고 dashboard_surface_envelope를 먼저 붙인다. 프론트가 envelope를 읽도록 옮긴 뒤에만 endpoint versioning 또는 root metadata 제거를 검토한다.
  • 보호 테스트: current endpoint JSON snapshots, frontend API contract tests, bootstrap initializing envelope, WS/SSE slice mapping parity.

6. JSONL writer contract

  • 공통 writer: per-path mutex, fresh fd, no poisoning, optional retention hook
  • 상위 store: audit, dated telemetry, goal events, trajectory는 domain schema만 소유
  • 목표: append correctness 설명과 구현을 하나의 substrate로 줄임.
  • 보호 테스트: append atomicity, dated mutex registry, corrupt line fallback, legacy audit fallback.

7. LSP-backed reverse engineering lane

  • 현재 존재: /api/v1/ide/lsp WS proxy와 dashboard IDE LSP client
  • 추가 목표: symbol graph, call graph, test owner를 추출해 이 HTML의 순서도를 자동 갱신
  • 설계 contract: docs/design/lsp-symbol-graph-exporter.md
  • 첫 구현: read-only exporter로 file -> symbol overview -> selected range -> references -> test owner를 JSON으로 반환
  • 보호 테스트: proxy handshake, workspace root scoping, no write capability, large-file timeout.

8. Refactor PR sequencing

  • PR 1: tests and snapshots only, no behavior change
  • PR 2: shared MCP request context reducer
  • PR 3: tool identity reducer and dispatcher boundary
  • PR 4: keeper TurnPlan extraction
  • PR 5: receipt/metrics close-out fixtures before moving post-turn code
  • PR 6: task/goal reducers and verification orphan recovery
  • PR 7: dashboard projection split and JSONL substrate cleanup
