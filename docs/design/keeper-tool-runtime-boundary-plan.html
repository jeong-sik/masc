Keeper Tool Runtime Boundary Plan

Goal-driven cleanup for the confusing Keeper_shell, Keeper_bash, GitHub, Docker, and Local execution split.

Updated 2026-05-24 Branch refactor/keeper-git-shell-bridge Worktree .worktrees/keeper-git-shell-bridge

Decision

Backend is not a tool surface

Docker or Local sandbox selection must be orthogonal to command semantics. Git, GitHub, typed Bash, shell inspection, and PR workflows should lower into backend-neutral requests before the runtime chooses Docker or Local execution.

Current Baseline

0 Old bad helper names in lib/keeper
0 Direct Docker runner calls outside Keeper_sandbox_runner
3,274 Combined LOC across main hot files and new sandbox helper modules

Slice 1 removes source-level references to Keeper_shell_docker, Keeper_shell_docker_exec_failure, and Keeper_shell_bash_docker. Slice 2 removes the legacy Keeper_shell_shared.run_docker* boundary and routes PR/GitHub tool execution through Keeper_sandbox_runner. Slice 3 moves keeper_shell op=gh compatibility execution into Keeper_shell_gh_bridge. Slice 4 moves keeper_shell op=git_clone compatibility execution into Keeper_shell_git_bridge. Slice 5 makes Keeper_sandbox_runner own route labels for bridge result shaping. Slice 6 moves read-side sandbox execution behind Keeper_sandbox_read_runner. Bridge and runner tests use injected mock runners.
