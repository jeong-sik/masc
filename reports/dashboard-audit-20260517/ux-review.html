Product Framing

MASC Dashboard should be judged as an operator console for long-running autonomous coding agents. It is not a generic SaaS dashboard and not a decorative cockpit. The primary UX promise is continuity: a human can see what each Keeper is doing, why it chose a tool/path, what proof exists, and what needs intervention.

UX Best Practices Applied

Human-AI Interaction

Set expectations, show uncertainty, allow correction, and support graceful failure. Apply this as: do not show AI advice as proof; distinguish evidence gaps from success.

Agentic AI Security

Continuous agents need least privilege, explicit accountability, action logging, human oversight, and reversibility. Apply this as: risk boundary panels and intervention queue.

Accessibility

Dense operator tools still need keyboard support, visible focus, semantic status messages, and non-color severity labels.

Observability

Logs alone are not enough. Trace/span/log correlation should explain root cause without manual grep across surfaces.

UX Recommendations

  1. Make Overview an action queue, not a decorative summary.
  2. Promote only surfaces that pass readiness parity and have a clear operator question.
  3. Render all status with source, timestamp, and proof completeness.
  4. Use a single evidence vocabulary: Satisfied, Missing evidence, Inconclusive, Failed, Advice.
  5. Use preview-confirm-recheck for all dashboard mutations.
  6. Make Code IDE line chips explain: Keeper, file/line, tool, decision, reason, trace, task, repo.
  7. Keep Lab and raw logs off the first screen; expose them through drilldowns/search.
