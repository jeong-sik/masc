Keeper Tool Runtime Boundary Plan

Goal-driven cleanup for the confusing Keeper_shell, Keeper_bash, GitHub, Docker, and Local execution split.

Updated 2026-05-25 Branch refactor/shell-ir-dispatch-axis-20260525 Worktree .worktrees/shell-ir-dispatch-axis-20260525

Decision

Backend is not a tool surface

Docker or Local sandbox selection must be orthogonal to command semantics. Git, GitHub, typed Bash, shell inspection, and PR workflows should lower into backend-neutral requests before the runtime chooses Docker or Local execution.

Current Baseline

0 Old bad helper names in lib/keeper
0 Direct Docker runner calls outside Keeper_sandbox_runner
0 Backend host exec calls using Keeper_shell actor

Slice 1 removes source-level references to Keeper_shell_docker, Keeper_shell_docker_exec_failure, and Keeper_shell_bash_docker. Slice 2 removes the legacy Keeper_shell_shared.run_docker* boundary and routes PR/GitHub tool execution through Keeper_sandbox_runner. Slice 5 removes the legacy keeper_shell op=gh and op=git_clone compatibility surface instead of preserving bridge shims. Slice 6 moves shell read-side sandbox execution behind Keeper_sandbox_read_runner. Slice 7 applies the same boundary to file tools. Slice 8 retires the tool-shaped Keeper_docker_read module name. Slice 9 moves backend host process execution to the sandbox backend actor. Slice 10 makes Keeper_shell_ir the Shell IR gate/path/dispatch center and Keeper_gh_runner the GH argv execution center. Slice 11 makes Masc_exec.Exec_program own the dev/read-only executable vocabulary. Slice 12 moves typed Bash input Shell IR construction behind Keeper_shell_ir. Slice 13 routes PR work-action command reads through Keeper_shell_command_semantics. Slice 14 routes nested-runtime guard-token extraction through Keeper_shell_command_words. Slice 15 routes approval action-key and history command-prefix extraction through Keeper_shell_command_words. Slice 16 routes Docker shell and GH raw command parsing through Keeper_shell_command_parse. Slice 17 pins keeper-wide raw parser and command-word ownership in boundary tests. Slice 18 moves GH repo slug/origin discovery into Keeper_gh_repo so Keeper_gh_shared stays parser/risk only. Slice 19 moves shell cwd/path/probe ownership into Keeper_shell_path and removes the stale shared direct host runner. Slice 20 moves shell op, timeout, and runtime-path ownership into dedicated modules. Slice 21 moves readonly hint/diagnosis ownership into Keeper_shell_readonly_policy and deletes Keeper_shell_shared. Slice 22 moves resource classification into Tool_resource_axis, consuming Keeper_tool_alias.canonical_resolution for public alias normalization before typed Bash executable lanes. Slice 23 removes substring fallback lane classification from Tool_resource_axis; non-catalog gated callers are explicit. Slice 24 adds Keeper_tool_capability_axis for semantic capability predicates. Slice 25 routes work-discovery and worktree-delta turn affordance tool lists through the same capability axis. Slice 26 moves the public alias projection into low-dependency Tool_name_alias_axis so coord required-tool matching and keeper alias routing share the same map. Slice 27 updates the Docker PR lifecycle reprobe harness to require public WebSearch/Bash surfaces plus native PR review tools instead of internal shell names. Runner tests use injected mock runners.
