최종 조립 경로

순서 주입 lane 무엇이 들어가나 대표 소스 현재 가시성
1 registry Config root, prompt markdown, runtime override를 로딩한다. Prompt_defaults.bootstrap_runtime, /api/v1/prompts Prompt Registry에서 일부 확인 가능.
2 system core behavior, profile policy, continuity, constitution, world, capabilities, identity/persona/goals. Keeper_prompt.build_keeper_system_prompt 최종 조립표 없음. 파일별로 흩어짐.
3 user/world unified system framing, turn intent, current world state, claimable work, board activity. Keeper_unified_prompt.build_prompt Prompt telemetry는 있으나 provenance 없음.
4 extra_system_context continuity snapshot, durable memory, skill route, worktree, telemetry feedback, turn instruction. Keeper_turn.build_turn_prompt 가장 중요한데 operator UI에서 거의 안 보임.
5 OAS hook temporal summary, claimed-task nudge, retry hint, Memory OS recall, tool filter/choice. Keeper_run_tools_hooks.before_turn_params digest/size 중심. 사람이 읽는 조립표 없음.
6 manifest context_injected edge와 prompt metrics가 기록된다. Keeper_agent_run.append_manifest edge는 있으나 segment provenance가 부족.

Stale tool guidance 위험

핵심 문제: tool 사용법이 prompt, live .masc config, keeper TOML, tool hints에 중복되어 있고 일부는 실제 public alias나 argument shape와 어긋난다. 모델 성격 변화처럼 보이는 현상 중 일부는 이 지침 drift로 설명된다.
패턴 관찰된 예 위험 정리 방향
Retired alias keeper_board_get vs keeper_board_post_get 없는 도구 호출, 같은 실패 반복. active schema/public alias 기준 linter.
Stale args keeper_task_done { notes } vs { task_id, result } 완료 보고 실패, task oscillation. keeper.tool_hints.toml을 schema example SSOT로 제한.
Conflicting PR flow keeper_pr_create 지침과 Execute typed argv 지침 공존 모델이 금지/허용 경계를 혼동. Keeper TOML에는 persona만, 공통 tool grammar 제거.
Hardcoded path repos/masc/lib/foo.ml worktree-first 정책 위반, stale path probe. repos/REPO_NAME/.worktrees/TASK_NAME 예시로 통일.

첫 PR 개선 목표

  1. Keeper detail 화면에 aggressive prompt assembly 패널을 추가한다.
  2. Prompt Registry 상단에도 Keeper assembly/stale guidance 요약을 노출한다.
  3. 각 row는 stage, lane, source prompt, source type, override 상태, byte size, risk를 보여준다.
  4. stale alias/args/path/PR-flow warning을 prompt effective text에서 즉시 스캔한다.
  5. 다음 PR에서 runtime manifest에 segment provenance를 추가할 수 있도록 UI 용어를 고정한다.

Done 기준

  • Override가 최종 prompt에 영향을 주는지 source badge로 바로 보인다.
  • Live .masc prompt가 stale tool guidance를 품으면 dashboard가 경고한다.
  • HTML audit artifact가 PR에 포함되어 review context를 제공한다.
